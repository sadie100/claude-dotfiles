# cafe24-figma-validate — Figma 시안 대조 검증 + 자동 수정

Figma 시안과 실제 페이지의 대상 컴포넌트를 **체크리스트** 기반으로 대조하고, 불일치가 있으면 시안과 일치할 때까지 수정하는 스킬이다. 검증은 Chrome에서, 수정은 cafe24-update 워크플로로, 마지막에 Safari 크로스체크를 한다.

## 0. 인자 검증 — 둘 다 없으면 즉시 실패

필수 인자 두 개를 반드시 받아야 한다:

1. **Figma 노드 링크** (`node-id` 포함)
2. **테스트 URL** — 검증 대상 페이지

인자가 없으면 **아무 것도 시작하지 말고 즉시 실패로 종료**하고 인자 형식을 안내한다. 기억이나 이전 대화로 URL·링크를 만들어내지 않는다 — Cafe24 테스트 스킨 번호는 스킨 복사 때마다 바뀌어서, 추측한 URL은 엉뚱한 스킨을 검증하게 된다.

선택 인자: **작업설명** — 페이지의 어느 컴포넌트가 대상인지 좁히는 용도. 없으면 Figma 노드의 내용(레이어명, 스크린샷)으로 대상을 추정하되, 페이지에서 특정이 안 되면 AskUserQuestion으로 확인한다.

## 1. 준비

- 브라우저는 **chrome-devtools-shared MCP**를 쓴다 (works/CLAUDE.md 공통 규칙 — 공유 크롬, 내가 연 탭에서만 작업).
- **뷰포트**: Figma 프레임 폭으로 데스크탑/모바일을 판단한다. 프로젝트 CSS의 브레이크포인트 **경계값에 걸치는 폭은 피한다** — 데스크탑 1280px, 모바일 375px 권장.
- 스크린샷은 **scratchpad 디렉토리**에 저장한다. 리포지토리에 저장 금지.
- **배포 상태 확인**: 로컬 최신 코드에만 있는 셀렉터/규칙이 라이브 CSS에 존재하는지 evaluate로 확인한다. 없으면 검증 결과가 옛 코드 기준이 된다 — 사용자에게 배포를 요청하거나, 로컬 수정분을 주입해 기준 상태를 만든 뒤 진행한다.

## 2. Figma 스펙 → 체크리스트

`get_design_context` + `get_screenshot`으로 대상 컴포넌트의 수치를 뽑아 **체크리스트**를 만든다. 항목: 폰트(크기/굵기/행간), 색상(텍스트/배경/보더), 간격(패딩/마진/gap), 크기, 라운드, 정렬. 이 체크리스트가 이후 모든 대조의 단일 기준이다.

## 3. 대조

페이지에서 대상 컴포넌트를 특정한 뒤:

1. **수치 대조** — evaluate로 `getComputedStyle` + `getBoundingClientRect`를 측정해 체크리스트 항목별로 기록. 허용 오차: 길이 ±1px, 색상은 정확 일치, 폰트 패밀리는 fallback 감안.
2. **육안 대조** — 스크린샷을 찍고 **Read로 직접 열어** Figma 스크린샷과 나란히 비교한다. 수치가 전부 일치해도 시각적으로 다를 수 있다 — 찍기만 하고 안 보는 것은 대조가 아니다.

**완료 기준: 체크리스트의 모든 항목이 일치 / 불일치 / 검증불가 중 하나로 기록됨.** 항목을 건너뛴 채 다음 단계로 가지 않는다.

## 4. 불일치 시 수정 — cafe24-update로 위임

불일치가 하나라도 있으면 **cafe24-update 스킬을 호출**해 수정한다 (라이브 주입으로 UI 확정 → 로컬 코드 반영). 파일부터 고치고 새로고침으로 확인하려 하지 말 것 — HMR이 없어서 그 순서는 성립하지 않으며, 함정 대응(cascade, 주입 위치, 삭제 시뮬레이션)은 전부 cafe24-update에 있다.

수정 확정 후 §3 체크리스트를 **재대조**한다. 모든 항목이 일치할 때까지 반복하되, 같은 항목이 3회 수정에도 안 잡히면 중단하고 원인 추정과 함께 보고한다 (예: 폰트 미탑재, 플랫폼 강제 스타일).

## 5. Safari 크로스체크

수정이 확정된 뒤(또는 처음부터 불일치가 없었으면 바로):

- 세션에 **safari-mcp-stp** MCP 툴이 있으면 Safari로 같은 URL을 열어 스크린샷을 찍고 Read로 열어 Chrome 결과와 육안 비교한다. 주의 깊게 볼 것: 폰트 렌더링, flex/gap 지원 차이, 스크롤바 폭, 이미지 비율.
- Safari에서만 깨지는 항목이 발견되면 §4로 돌아가 수정하되, Chrome 결과를 깨뜨리지 않는지 양쪽에서 재확인한다.
- 툴이 없으면 우회하거나 다른 방법을 시도하지 말고, 보고에 "Safari 크로스체크 생략 — safari-mcp-stp가 이 프로젝트에 미등록 (`claude mcp add`로 등록 필요)"을 명시한다.

```markdown
| 배경색 | #F5F5F5 | #F5F5F5 | - | ✅ |

- 수정한 파일/위치: (있으면. "Cafe24 관리자 배포 필요" 안내 포함)
- Safari 크로스체크: (결과 / 생략 사유)

```

마무리로 리포지토리에 임시 파일이 생기지 않았는지 `git status`로 확인하고 정리한다.
